# intelligent-traffic-control-system

# Smart Crossroads Belarus (Адаптивная система управления дорожным трафиком)

Прототип асинхронного веб-сервиса для интеллектуального регулирования светофорных объектов на основе компьютерного зрения и прогнозной аналитики. Проект вдохновлен современными мировыми практиками внедрения умных транспортных систем (ITS), в частности архитектурой UTC-UX Fusion (Дубай), и адаптирован под инфраструктурные особенности Республики Беларусь.

---

## 📌 Описание проекта

Цель проекта — разработка программного комплекса, который заменяет жестко запрограммированные таймеры светофоров на динамическое (адаптивное) управление, подстраивающееся под реальную дорожную ситуацию в режиме реального времени.

Чтобы избежать избыточной перегрузки системы, рамки дипломного проекта ограничены **прототипом серверной логики управления одним сложным четырехсторонним перекрестком (или проспектом/улицей)**.

### Ключевые компоненты системы:
1. **Слой ИИ и Компьютерного зрения (Computer Vision):** Обработка видеопотока с дорожных камер (детектирование транспорта, оценка длины очередей в полосах, классификация спецтранспорта).
2. **Асинхронный Бэкенд (FastAPI):** Высокопроизводительный шлюз для приема метаданных от камер, расчета оптимальных фаз и передачи команд на контроллеры светофоров через WebSockets.
3. **Модуль приоритетного проезда:** Мгновенный перерасчет фаз («зеленый коридор») при обнаружении экстренных служб (скорая помощь, МЧС, милиция).
4. **Интерактивный Дашборд (WebUI):** Визуализация работы перекрестка, графиков эффективности ИИ и текущего состояния светофорных фаз для демонстрации на защите.

## 📅 Примерный план разработки (Дорожная карта проекта)

### Этап 1: Исследование и проектирование

* [ ] Анализ существующих решений и нормативных актов организации дорожного движения в РБ.
* [ ] Проектирование архитектуры веб-сервиса и схемы базы данных.

### Этап 2: Разработка ядра системы (Бэкенд и CV-модуль)

* [ ] Развертывание базового каркаса FastAPI, настройка роутинга и подключения к БД.
* [ ] Интеграция и базовая настройка модели для детекции автомобилей и спецтранспорта на тестовых видеозаписях.
* [ ] Написание скрипта-эмулятора, который имитирует отправку данных с камер на сервер.

### Этап 3: Реализация бизнес-логики и управления фазами

* [ ] Разработка алгоритма адаптивного изменения длительности сигналов (зеленой фазы) на основе загруженности направлений.
* [ ] Реализация асинхронного триггера «Зеленый коридор» для приоритетного транспорта.
* [ ] Настройка двустороннего обмена данными через WebSockets между сервером, эмулятором светофора и клиентом.

### Этап 4: Разработка интерфейса и визуализация

* [ ] Создание веб-панели для отображения схемы перекрестка и анимации движения потоков.
* [ ] Вывод графиков аналитики (среднее время ожидания, длина пробок, количество пропущенных машин).

### Этап 5: Тестирование, оптимизация и подготовка к защите

* [ ] Нагрузочное тестирование FastAPI-сервера.
* [ ] Сравнение эффективности адаптивного алгоритма с классическим фиксированным таймером (сбор метрик для исследовательской части диплома).
* [ ] Оформление пояснительной записки диплома.
