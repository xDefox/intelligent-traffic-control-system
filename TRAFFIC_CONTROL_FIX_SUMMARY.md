# Исправление алгоритма управления светофорами

## Проблема

Алгоритм управления светофорами работал странно:
1. **Конфликтующие зелёные сигналы**: На перекрёстке между 1 и 2 одновременно могли гореть зелёные сигналы на противоположных направлениях
2. **Не учитывалась загруженность**: Светофоры не реагировали на пробки на downstream дорогах
3. **Отсутствовала координация**: Каждый светофор работал независимо, без учёта соседних
4. **Слишком частые переключения**: Светофоры переключались раз в минуту, даже при пробках

## Причины

1. **Отсутствовала конфигурация перекрёстков**: В `road_config.py` не было определений перекрёстков с фазами
2. **Нет координации на уровне перекрёстка**: Каждый светофор (Dubai-style) работал независимо
3. **Не учитывался downstream**: Алгоритм не проверял, перегружены ли следующие перекрёстки
4. **Каскадная система была неполной**: Она только корректировала время, но не предотвращала конфликты

## Решения

### 1. Добавлена конфигурация перекрёстков (`backend/core/road_config.py`)

```python
"intersection_1": {
    "type": "T",
    "phases": {
        "NS": {"approaches": ["approach_2", "approach_3"]},  # Север-Юг
        "EW": {"approaches": ["approach_0", "approach_1"]},  # Восток-Запад
    },
},
"intersection_2": {
    "type": "X",
    "phases": {
        "NS": {"approaches": ["approach_2", "approach_3"]},
        "EW": {"approaches": ["approach_0", "approach_1"]},
    },
},
```

### 2. Реализована координация на уровне перекрёстка (`backend/services/traffic_brain.py`)

**Глобальное состояние фаз:**
```python
_intersection_phase_state: Dict[str, dict] = {}
# Хранит активную фазу для каждого перекрёстка
```

**Проверка разрешения фазы (`_is_phase_allowed`):**
- Отслеживает активную фазу на перекрёстке
- Не позволяет одновременно гореть противоречивым фазам (NS vs EW)
- Обеспечивает минимальное время горения фазы (5 секунд)
- Автоматически переключает фазы при необходимости

### 3. Добавлена проверка downstream загруженности (`_get_downstream_congestion`)

```python
def _get_downstream_congestion(intersection_id: str, lane_id: str) -> float:
    """
    Получить загруженность downstream дорог.
    Если downstream перегружен >80%, не пускаем новых машин.
    """
```

**Логика:**
- Получает downstream перекрёстки из графа дорог
- Считает среднюю загруженность всех полос downstream
- Если загруженность >80% → держит светофор красным
- Предотвращает заторы на перекрёстках

### 4. Улучшена логика принятия решений (`process_lane_telemetry`)

**Новый порядок проверок:**
1. Обновление состояния полосы
2. **Проверка координации фаз** (новое)
3. **Проверка downstream загруженности** (новое)
4. Принятие решения о цвете сигнала

**Пример работы:**
```
1. Машины на подходе 1 (восток) перекрёстка 1
2. Проверяем: фаза EW разрешена? → Да
3. Проверяем: downstream (перекрёсток 2) свободен? → Да (30% загруженность)
4. Включаем зелёный на 10 секунд (на основе загруженности 70%)
```

## Результат

### До исправления:
- ❌ Противоречивые зелёные сигналы на перекрёстке
- ❌ Пробки на downstream дорогах игнорировались
- ❌ Светофоры переключались каждую минуту
- ❌ Нет координации между перекрёстками

### После исправления:
- ✅ Только одна фаза горит на перекрёстке (NS или EW)
- ✅ Downstream загруженность учитывается (>80% → красный)
- ✅ Адаптивное время зелёного (5-30 сек на основе загруженности)
- ✅ Координация между перекрёстками через каскадные команды
- ✅ Минимальное время горения фазы (5 сек) предотвращает слишком частые переключения

## Логирование

Новый алгоритм выводит информативные сообщения:
```
🟢 [intersection_1_approach_1] Машины: 7/10 (70%) → GREEN на 22.5с
🔴 [intersection_1_approach_0] Фаза EW заблокирована конфликтной → RED
🔴 [intersection_2_approach_0] Downstream перегружен (85%) → RED
🔄 [intersection_1] Переключение фазы: NS → EW
```

## Файлы изменены

1. `backend/core/road_config.py` - добавлены определения перекрёстков
2. `backend/services/traffic_brain.py` - добавлены:
   - Глобальное состояние фаз `_intersection_phase_state`
   - Метод `_is_phase_allowed()` - координация фаз
   - Метод `_get_downstream_congestion()` - проверка downstream
