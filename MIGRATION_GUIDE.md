# Руководство по миграции на новую систему Waypoints

## Обзор изменений

### Старая система (RoadSegment-based):
- ❌ Waypoints разбиты на сегменты (RoadSegment)
- ❌ Последний waypoint сегмента A ≠ первому waypoint сегмента B
- ❌ Дерганье при переходе между сегментами
- ❌ Сложная логика застревания
- ❌ Машины застревают на перекрёстках

### Новая система (Global Waypoint Graph):
- ✅ Единый граф waypoints (WaypointNode)
- ✅ Waypoints совмещены на стыках сегментов
- ✅ Плавные переходы без дерганья
- ✅ Упрощённая логика движения
- ✅ Нет застреваний на перекрёстках

## Шаг 2: Создание WaypointNode объектов

### 2.1 Создайте пустые GameObject для waypoints

**Для прямых дорог (1 туда / 1 обратно):**
```
Создайте 5-10 waypoints на каждой дороге:
- Назовите: "WP_North_1", "WP_North_2", ..., "WP_North_5"
- Расставьте их через равные интервалы (5-10м)
- Направьте в сторону движения

Повторите для противоположного направления:
- "WP_South_1", "WP_South_2", ...
```

**Для перекрёстка (4 направления × 3 варианта):**

Пример для перекрёстка с 4 подъездами:

```
Подъезд с Севера (3 waypoint перед перекрёстком):
- WP_North_1 (спавн)
- WP_North_2
- WP_North_3 (перед стоп-линией)

Подъезд с Юга:
- WP_South_1
- WP_South_2
- WP_South_3

Подъезд с Запада:
- WP_West_1
- WP_West_2
- WP_West_3

Подъезд с Востока:
- WP_East_1
- WP_East_2
- WP_East_3
```

### 2.2 Добавьте компонент WaypointNode

На каждый waypoint GameObject:
1. Нажмите **Add Component**
2. Найдите **Waypoint Node**
3. Настройте параметры:
   - `isIntersection`: `true` для waypoints на перекрёстке (для визуализации жёлтым цветом)
   - `gizmoColor`: оставьте cyan для обычных waypoints

**Примечание:** WaypointNode содержит ТОЛЬКО список соседей. Все параметры движения (скорость, остановка) управляются WaypointNavigator и светофорами.

## Шаг 3: Связывание waypoints (Neighbours)

### 3.1 Прямые дороги

**Пример: Дорога с Севера на Юг через перекрёсток**

```
Структура:
WP_North_1 → WP_North_2 → WP_North_3 → [Intersection_North] → WP_South_1 → WP_South_2 → WP_South_3

Настройка:
1. Выберите WP_North_1
   - В списке Neighbours перетащите ТОЛЬКО WP_North_2 (вперёд по движению)
   - ⚠️ НЕ добавляйте обратную связь! Только 1 сосед.

2. Выберите WP_North_2
   - Neighbours: ТОЛЬКО WP_North_3

3. Выберите WP_North_3
   - Neighbours: ТОЛЬКО [Intersection_North]

4. Выберите [Intersection_North]
   - Neighbours: WP_South_1, WP_West_1, WP_East_1 (все возможные направления)
   - isIntersection: true

5. Выберите WP_South_1
   - Neighbours: ТОЛЬКО WP_South_2

6. И так далее...

⚠️ КРИТИЧЕСКИ ВАЖНО:
- На прямых дорогах у каждого waypoint должен быть ТОЛЬКО 1 сосед (следующий waypoint вперёд)
- НЕ добавляйте двусторонние связи (WP1→WP2 И WP2→WP1) - это вызовет развороты!
- На развилках/перекрёстках: все возможные направления (3+ соседей)
```

### 3.2 Перекрёсток (4 направления × 3 варианта)

**Пример: Перекрёсток с возможными поворотами**

```
Въезд с Севера:
WP_North_1 → WP_North_2 → [Intersection_North_Entry]

[Intersection_North_Entry] имеет 3 соседа:
- [Turn_Left_WP] (поворот налево на Запад)
- [Straight_WP] (прямо на Юг)
- [Turn_Right_WP] (поворот направо на Восток)

Въезд с Запада:
WP_West_1 → WP_West_2 → [Intersection_West_Entry]

[Intersection_West_Entry] имеет 3 соседа:
- [Turn_Left_WP] (поворот налево на Юг)
- [Straight_WP] (прямо на Восток)
- [Turn_Right_WP] (поворот направо на Север)

ВАЖНО: [Turn_Left_WP] для въезда с Севера = [Turn_Left_WP] для въезда с Востока
Это ОДНА И ТА ЖЕ точка в пространстве!
```

### 3.3 Используйте контекстное меню для автоматизации

На каждом WaypointNode можно использовать:
- **Context Menu → "Найти nearby waypoints"** - автоматически найдёт waypoints в радиусе 5м
- **Context Menu → "Очистить соседей"** - очистит список соседей

⚠️ **Внимание:** Автоматическое нахождение - только для помощи, всегда проверяйте связи вручную!

## Шаг 4: Настройка TrafficGenerator

### 4.1 Обновите Spawn Routes

В инспекторе TrafficGenerator:
1. Разверните **Spawn Routes**
2. Для каждого маршрута:
   - `Spawn Point`: перетащите точку спавна (как было)
   - `Start Waypoint`: перетащите **первый waypoint** дороги (новое поле)

**Пример:**
```
Spawn Routes:
0. "North Approach"
   - Spawn Point: SpawnPoint_North
   - Start Waypoint: WP_North_1

1. "South Approach"
   - Spawn Point: SpawnPoint_South
   - Start Waypoint: WP_South_1
```

### 4.2 Проверьте настройки
- `Spawn Interval`: 3 сек (по умолчанию)
- `Max Active Cars`: 50 (по умолчанию)
- `Show Road Gizmos`: true (для отладки)

---

## Шаг 5: Настройка светофоров (без изменений)

### 5.1 Светофоры остаются как есть

Светофоры работают через триггеры:
- У каждого перекрёстка есть **StopTrigger** (триггер с тегом "StopTrigger")
- WaypointNavigator автоматически детектирует светофор через `OnTriggerStay`
- Никаких изменений в светофорах не требуется!

### 5.2 Проверьте триггеры
Убедитесь что:
- ✅ На перекрёстке есть триггеры с тегом "StopTrigger"
- ✅ Триггеры покрывают зону остановки перед светофором
- ✅ На триггерах висит компонент TrafficLightViewer

## Шаг 6: Тестирование

### 6.1 Запустите сцену

1. Нажмите **Play**
2. Машины должны спавниться на точках спавна
3. Машины едут по waypoints к перекрёстку
4. На перекрёстке останавливаются на красный свет
5. При зелёном проезжают перекрёсток
6. Выбирают случайное направление на развилке

### 6.2 Проверьте в консоли

Должны появляться логи:
```
[Car_1] Завершил объезд (если было застревание)
[Car_2] Нет доступных соседей, уничтожаем (если доехал до конца)
```

### 6.3 Визуальная отладка

Включите **Gizmos** в окне Scene:
- 🔵 Cyan sphere = обычный waypoint
- 🟡 Yellow sphere = перекрёсток/развилка
- 🟢 Зелёные линии = связи между waypoints
- 🔴 Красный луч = детекция машин впереди
- 🟡 Жёлтая линия = текущий путь машины к waypoint

## Шаг 7: Распространённые проблемы

### Проблема: Машина не едет
**Решение:**
- Проверьте что у машины есть WaypointNavigator компонент
- Проверьте что SetupNode() был вызван с валидным WaypointNode
- Проверьте что у WaypointNode есть neighbours

### Проблема: Машина застревает на перекрёстке
**Решение:**
- Убедитесь что waypoint на перекрёстке имеет isIntersection = true
- Проверьте что у перекрёстка есть минимум 2 соседа (выезды)
- Проверьте что светофор работает (триггеры на месте)

### Проблема: Машина дергается
**Решение:**
- Проверьте что waypoints совмещены (одна точка для въезда и выезда)
- Увеличьте rotationSpeed (сейчас 15, попробуйте 20-25)
- Уменьшите speedSmoothing (сейчас 5, попробуйте 3-4)

### Проблема: Машины проезжают друг через друга
**Решение:**
- Увеличьте maxCheckDistance (сейчас 2.2м, попробуйте 3-4м)
- Увеличьте emergencyBrakeDistance (сейчас 1.5м, попробуйте 2-2.5м)
- Убедитесь что у машин есть Collider и тег "Car"

## Шаг 8: Оптимизация

### 8.1 Настройка параметров

**Для высокой интенсивности (много машин):**
```csharp
speed = 15f;                    // Быстрее едут
maxCheckDistance = 3f;          // Дальше видят
emergencyBrakeDistance = 2f;    // Раньше тормозят
spawnInterval = 2f;             // Чаще спавнятся
maxActiveCars = 80;             // Больше машин
```

**Для низкой интенсивности (мало машин):**
```csharp
speed = 10f;                    // Медленнее
maxCheckDistance = 2f;          // Меньше дистанция
emergencyBrakeDistance = 1.5f;  // Стандарт
spawnInterval = 5f;             // Реже спавнятся
maxActiveCars = 30;             // Меньше машин
```

### 8.2 Производительность

- Уменьшите `maxActiveCars` если FPS падает
- Увеличьте `spawnInterval` для меньшей нагрузки
- Отключите `ShowDebugGizmos` в продакшене

## Пример полной конфигурации перекрёстка

```
Перекрёсток "Intersection_1" с 4 подъездами:

Въезд с Севера (поворот налево/прямо/направо):
- WP_North_1 (spawn) → neighbours: [WP_North_2]
- WP_North_2 → neighbours: [WP_North_3]
- WP_North_3 → neighbours: [Intersection_North_Entry]
- Intersection_North_Entry (isIntersection=true) → neighbours: [Turn_Left_WP, Straight_WP, Turn_Right_WP]

Въезд с Юга:
- WP_South_1 → neighbours: [WP_South_2]
- WP_South_2 → neighbours: [WP_South_3]
- WP_South_3 → neighbours: [Intersection_South_Entry]
- Intersection_South_Entry (isIntersection=true) → neighbours: [Turn_Left_WP, Straight_WP, Turn_Right_WP]

Общие waypoints на перекрёстке:
- Turn_Left_WP (поворот налево) → neighbours: [WP_West_1] (для въезда с Севера)
                                    → neighbours: [WP_South_1] (для въезда с Востока)
                                    → neighbours: [WP_East_1] (для въезда с Юга)
                                    → neighbours: [WP_North_1] (для въезда с Запада)

- Straight_WP (прямо) → neighbours: [WP_South_1] (для въезда с Севера)
                        → neighbours: [WP_North_1] (для въезда с Юга)
                        → neighbours: [WP_East_1] (для въезда с Запада)
                        → neighbours: [WP_West_1] (для въезда с Востока)

- Turn_Right_WP (поворот направо) → neighbours: [WP_East_1] (для въезда с Севера)
                                   → neighbours: [WP_West_1] (для въезда с Юга)
                                   → neighbours: [WP_North_1] (для въезда с Востока)
                                   → neighbours: [WP_South_1] (для въезда с Запада)
```
